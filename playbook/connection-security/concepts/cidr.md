# CIDR (Classless Inter-Domain Routing)

## Overview

CIDR is a method of representing IP networks using prefix notation rather than traditional address classes. It allows networks to be divided into appropriately sized subnets, making IPv4 address allocation much more efficient.

## Key Concepts

- Prefix length
- Slash notation (/24)
- Network bits
- Host bits
- Variable-length subnetting

## Why It Matters

CIDR is used throughout modern networking. Security professionals encounter CIDR when configuring firewalls, VPNs, cloud networks, routing tables, and access control lists.

## Common Uses

- Network design
- Firewall configuration
- Cloud networking
- VPN configuration
- Routing

## Best Practices

- Memorize common prefixes such as /24, /25, /26, /27, /16, and /8.
- Understand how increasing the prefix decreases available hosts.
- Be comfortable converting between CIDR notation and subnet masks.
