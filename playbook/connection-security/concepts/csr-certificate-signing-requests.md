# Certificate Signing Requests (CSRs)

## Overview

A Certificate Signing Request (CSR) is a message generated by an individual, device, or organization to request a digital certificate from a Certificate Authority (CA).

Contains identifying information and the requester's public key, which the CA uses to issue a certificate after verifying the request.

A CSR does not contain the private key.

## Key Concepts

- Public Key
- Private Key
- Certificate Authority (CA)
- Digital Certificate
- Identity Verification

## Why It Matters

CSRs provide a secure way to request digital certificates without exposing private keys. They are a critical step in the Public Key Infrastructure (PKI) certificate issuance process.

## Common Uses

- HTTPS Certificates
- TLS
- VPN Certificates
- Secure Email
- Internal PKI

## Best Practices

- Generate the private key securely and keep it confidential.
- Never include or share the private key in a CSR.
- Provide accurate identifying information.
- Submit CSRs only to trusted Certificate Authorities.
- Protect the issued certificate and its corresponding private key.
