# Man-in-the-Middle (MitM)

## Overview

A Man-in-the-Middle (MitM) attack occurs when an attacker secretly intercepts and possibly alters communications between two parties without either side realizing it.

## Key Concepts

- Interception
- Eavesdropping
- Session interception
- Traffic manipulation
- Authentication

## Why It Matters

MitM attacks can expose credentials, sensitive data, and communications. Proper encryption and certificate validation greatly reduce this risk.

## Common Uses

- Credential theft
- Traffic interception
- Data manipulation
- Session compromise

## Best Practices

- Use HTTPS and TLS.
- Verify certificates.
- Avoid unsecured public Wi-Fi.
- Implement mutual authentication when appropriate.
