# Subnetting

## Overview

Subnetting divides a larger IP network into smaller logical networks. It improves performance, reduces broadcast traffic, and increases security by isolating systems into separate network segments.

## Key Concepts

- Network ID
- Broadcast address
- Host range
- Subnet mask
- CIDR prefix
- Default gateway

## Why It Matters

Subnetting is used in nearly every enterprise environment. Security professionals use subnetting to understand network boundaries, configure firewalls, troubleshoot connectivity, and perform accurate vulnerability assessments.

## Common Uses

- Network segmentation
- VLAN design
- Firewall configuration
- Access control
- Enterprise network design

## Best Practices

- Never assign the network address to a host.
- Never assign the broadcast address to a host.
- Verify subnet boundaries before scanning a network.
- Design subnets according to organizational and security needs.
