# Virtual Private Network (VPN)

## Overview

A Virtual Private Network (VPN) creates an encrypted tunnel between a device and another network over an untrusted network such as the Internet. By encrypting traffic, a VPN protects data from being intercepted while it is in transit.

VPNs are commonly used to provide secure remote access and securely connect separate networks.

## Key Concepts

### Encrypted Tunnel

A VPN establishes a secure, encrypted connection known as a tunnel.

All data sent through the tunnel is encrypted before transmission and decrypted upon arrival, protecting it from eavesdropping and tampering.

### Remote Access VPN

A remote access VPN allows individual users to securely connect to a private network from another location.

Common uses include:

- Working from home
- Accessing internal company resources
- Secure administration of remote systems

### Site-to-Site VPN

A site-to-site VPN securely connects two or more separate networks over the Internet.

Organizations commonly use site-to-site VPNs to connect branch offices without dedicated leased lines.

### Authentication

Before a VPN connection is established, users or devices must authenticate.

Authentication may use:

- Username and password
- Digital certificates
- Multi-Factor Authentication (MFA)
- Pre-shared keys (PSKs)

### VPN Protocols

Common VPN protocols include:

- IPsec
- OpenVPN
- WireGuard
- SSL/TLS VPNs

Each protocol provides encrypted communication but differs in performance, configuration, and implementation.

## Why It Matters

VPNs protect sensitive information as it travels across untrusted networks. They are essential for secure remote work, connecting geographically separated offices, and protecting communications from interception.

## Common Uses

- Secure remote access
- Connect branch offices
- Protect data on public Wi-Fi
- Secure administrative access
- Encrypt network traffic
- Protect sensitive communications

## Best Practices

- Use modern VPN protocols and strong encryption.
- Require Multi-Factor Authentication whenever possible.
- Keep VPN software and appliances updated.
- Disable weak or outdated encryption algorithms.
- Monitor VPN logs for unusual authentication or connection activity.
