# SHA-256

## Overview

SHA-256 (Secure Hash Algorithm 256-bit) is a cryptographic hash algorithm that converts data into a fixed-length 256-bit hash value. It is a one-way function, meaning the original data cannot be recreated from the hash.

SHA-256 is one of the most widely used hashing algorithms in modern cybersecurity.

## Key Concepts

- Cryptographic Hash Function
- 256-bit Hash
- One-Way Function
- Data Integrity
- Collision Resistance

## Why It Matters

SHA-256 allows systems to verify that data has not been modified. Even a one-character change produces a completely different hash, making it an effective tool for detecting tampering.

## Common Uses

- Password Storage
- Digital Signatures
- File Integrity Monitoring
- Software Verification
- Blockchain
- Digital Certificates

## Best Practices

- Use SHA-256 or stronger algorithms for security-sensitive applications.
- Use hashing to verify integrity, not to encrypt data.
- Avoid deprecated algorithms such as MD5 and SHA-1 for modern security.
- Verify hashes when downloading important software.
