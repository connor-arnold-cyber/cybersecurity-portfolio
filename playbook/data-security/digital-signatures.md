# Digital Signatures

## Overview

A digital signature uses asymmetric cryptography to verify the authenticity, integrity, and origin of digital information.

A digital signature does not encrypt data. Instead, it proves who created it and that it has not been modified.

## Key Concepts

- Public Key
- Private Key
- Hashing
- Authentication
- Non-Repudiation

## Why It Matters

Digital signatures allow recipients to verify that a message truly came from the sender and has not been altered during transmission.

## Common Uses

- Signed Software
- Email Security
- Code Signing
- Digital Certificates
- Financial Transactions

## Best Practices

- Protect private keys.
- Verify signatures before trusting files.
- Use trusted Certificate Authorities.
